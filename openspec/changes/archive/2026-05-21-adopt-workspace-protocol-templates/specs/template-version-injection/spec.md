## ADDED Requirements

### Requirement: Templates use workspace protocol for internal packages
All project templates (`template-react-ts`, `template-react-js`, `template-vue3-ts`, `template-vue3-js`) SHALL declare internal `@devkit/*` package dependencies using `workspace:^` protocol in `package.json.ejs`. Hardcoded version ranges (e.g. `^0.0.1`, `^1.0.0`) SHALL NOT appear in template source for any internal `@devkit/*` package.

#### Scenario: react-ts template uses workspace protocol
- **WHEN** inspecting `packages/devkit-plugin-react/templates/template-react-ts/package.json.ejs`
- **THEN** the file SHALL contain `"@devkit/service": "workspace:^"` and similar entries for any other `@devkit/*` package
- **AND** the file SHALL NOT contain any hardcoded version like `"@devkit/service": "^0.0.1"`

#### Scenario: All four templates consistent
- **WHEN** scanning all 4 template `package.json.ejs` files
- **THEN** every `@devkit/*` dependency SHALL use `workspace:^` literal as the version

### Requirement: CLI normalizes workspace protocol after rendering
The CLI `create` command SHALL execute a `normalizeDeps` step immediately after `renderTemplates`, replacing every occurrence of `workspace:^` in the generated project's `package.json` with a concrete version range. The final `package.json` written to disk SHALL NOT contain any `workspace:^` literal.

#### Scenario: workspace protocol replaced in monorepo mode
- **WHEN** CLI runs in monorepo mode (`depMode.kind === 'link'`)
- **THEN** every `@devkit/<pkg>: workspace:^` SHALL be replaced with `@devkit/<pkg>: link:<absolute-path-to-monorepo>/packages/devkit-<pkg>`

#### Scenario: workspace protocol replaced in npm mode
- **WHEN** CLI runs in npm mode (`depMode.kind === 'npm'`)
- **THEN** every `@devkit/<pkg>: workspace:^` SHALL be replaced with `@devkit/<pkg>: ^${cliVersion}`

#### Scenario: No workspace literal leaks to user
- **WHEN** any project is generated by `dc create` regardless of mode
- **THEN** `cat <project>/package.json | grep workspace:` SHALL produce zero matches

### Requirement: Dependency mode resolution
The CLI SHALL resolve a dependency mode (`IDepMode`) before rendering templates, using the following priority order: explicit environment variable, monorepo auto-detection, npm fallback. The resolution SHALL be deterministic given the same `cwd` + environment.

#### Scenario: Monorepo auto-detection
- **WHEN** CLI is invoked with cwd inside a directory tree containing both `pnpm-workspace.yaml` and `packages/devkit-service/`
- **THEN** `resolveDepMode` SHALL return `{ kind: 'link', monorepoRoot: <detected-root>, cliVersion: <version> }`

#### Scenario: Outside monorepo
- **WHEN** CLI is invoked with cwd not inside any devkit monorepo
- **THEN** `resolveDepMode` SHALL return `{ kind: 'npm', cliVersion: <version> }`

#### Scenario: Environment variable override
- **WHEN** `DEVKIT_DEP_MODE=npm` is set
- **THEN** `resolveDepMode` SHALL return `{ kind: 'npm', cliVersion: <version> }` regardless of cwd

#### Scenario: Explicit monorepo root via env
- **WHEN** `DEVKIT_DEP_MODE=link` and `DEVKIT_MONOREPO_ROOT=/path/to/monorepo` are both set
- **THEN** `resolveDepMode` SHALL return `{ kind: 'link', monorepoRoot: '/path/to/monorepo', cliVersion: <version> }`

#### Scenario: Bundler injection respects mode
- **WHEN** `addBundlerToDevDeps(targetDir, 'vite', depMode)` is called with `depMode.kind === 'link'`
- **THEN** the written `devDependencies['@devkit/bundler-vite']` SHALL be `link:<monorepoRoot>/packages/devkit-bundler-vite`

## MODIFIED Requirements

### Requirement: Complete creation flow
The `create` command SHALL execute the full flow: validate name → select template → select bundler → select package manager → enter description → render templates → normalize dependency versions → write bundler to devDependencies → install dependencies (using selected PM) → invoke framework generator → print success message with PM-aware startup commands.

The `normalize dependency versions` step SHALL replace every `workspace:^` literal in the generated `package.json` with either a `link:` URI (monorepo mode) or a `^${cliVersion}` range (npm mode), so the resulting file is consumable by any package manager outside a pnpm workspace.

#### Scenario: Full creation flow succeeds in monorepo
- **WHEN** user runs `pnpm exec dc create my-app -t react-ts -b vite --pm pnpm` from inside the devkit monorepo
- **THEN** files SHALL be generated in `./my-app`
- **AND** `my-app/package.json` SHALL contain `link:` URIs for every `@devkit/*` dependency
- **AND** `pnpm install --ignore-workspace` in `my-app` SHALL succeed
- **AND** the success message SHALL include `cd my-app && pnpm dev`

#### Scenario: Full creation flow with all interactive prompts
- **WHEN** user runs `dc create my-app` and answers all prompts
- **THEN** the same dependency normalization SHALL apply
- **AND** generated `package.json` SHALL NOT contain any `workspace:^` literal
